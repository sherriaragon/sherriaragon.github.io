Overview

This website is my presentation. To view the code for this site go here: https://github.com/sherriaragon/sherriaragon.github.io/ and click on any HTML or CSS page. I decided to use this format because the information is too long and complex for a PowerPoint or Google Slides presentation. Building the website and creating the content is much easier because you can see the code and how it renders on the actual website instead of seeing screenshots of the website and code in a PowerPoint presentation.

This site uses nested containers that are styled differently to give the appearance of layers. The Homepage for this site is a series of images and links to subsequent pages and other websites typical of any other Homepage. The Homepage is a three-column two-row layout. The images are linked to book summaries. The text for the book summaries is just filler text (because structure is the point, not content in this case). The nav bar is linked to the full stories published on Online-literature.com as the copyrights have expired. Only the first word of the book title after the article "The" is used so that users know which book is linked. I created the images for this website because book publishers recreate book covers and copyright them.

My how-to guide for building an HTML5 website with external CSS is linked. View the "Overview" on Page 1 in the guide to know some of the differences between HTML5 and HTML.

HTML5 is the 5th version of the website authoring language, Hypertext Markup Language. HTML5 incorporates the functionality of all previous versions of HTML while integrating new technologies such as text-to-speech, speech-to-text, video, 3-D graphics and more. HTML5 defines structure, layout, and functionality using characters contained within opening and closing tags that have attributes telling browsers how to render the webpage. External CSS is included in this guide in order to separate content from formatting as recommended by the W3C Style Guide.

The World Wide Web Consortium (W3C) is the standards body who decides, publishes, and implements HTML and CSS rules worldwide. Its purpose is to expand website accessibility to more audiences by standardizing web development rules. Adhering to W3C rules increases search engine optimization (SEO) and website visits (theoretically).

HTML5 is open-source and freely accessible to people willing to learn it regardless of socioeconomic background, culture, language, geographical location, technological skill or physical impairment.
